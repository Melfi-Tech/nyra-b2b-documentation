## Overview

Creates a new wallet for a business customer. This endpoint allows businesses to create sub-wallets for their customers with specified parameters.

## Authentication

- **Type**: Bearer Token
- **Headers**:
  - `Authorization`: `Bearer <jwt_token>` (required, string)

## Prerequisites

- User must be authenticated with valid JWT token
- User must have permission to create wallets for the business

## Request Body

```json
{
  "customer_id": "string",
  "wallet_type": "string",
  "currency": "string",
  "description": "string"
}
```

## Response

Returns the created wallet object with details including wallet ID, customer ID, type, currency, and status.

## Error Handling

- **400**: Invalid request parameters
- **401**: Invalid or expired token
- **403**: User lacks permission to create wallets
- **409**: Wallet already exists for customer
