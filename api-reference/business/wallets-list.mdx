## Overview

Retrieves all customer sub-wallets for the business associated with the authenticated API client. This endpoint excludes business float wallets and provides a comprehensive list of customer accounts.

## Authentication

- **Type**: API Client
- **Headers**:
  - `x-client-id`: Client ID (required, string)
  - `Authorization`: `Bearer <client_secret>` (required, string)

## Prerequisites

- API client must be authenticated and authorized for the business
- Business must have existing customer wallets

## Response

Returns an array of wallet objects with detailed information including account numbers, bank names, and ownership details.

## Error Handling

- **401**: Invalid client ID or secret
- **403**: API client not authorized for the business
