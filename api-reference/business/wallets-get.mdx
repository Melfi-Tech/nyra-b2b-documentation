## Overview

Retrieves details of a specific sub-wallet (customer or float) for the business associated with the authenticated API client.

## Authentication

- **Type**: API Client
- **Headers**:
  - `x-client-id`: The client ID (required, string)
  - `Authorization`: `Bearer <client_secret>` (required, string)

## Path Parameters

- `walletId`: The ID of the wallet (required, string)

## Response

### Success Response (200 OK)

```json
{
  "success": true,
  "message": "Wallet fetched successfully",
  "data": {
    "wallet_id": "string",
    "account_number": "string",
    "bank_name": "string",
    "owners_fullname": "string",
    "frozen": false,
    "business_id": "string",
    "isFloat": false,
    "created_at": "string",
    "updated_at": "string"
  }
}
```

## Error Handling

- **400 Bad Request**: Invalid wallet ID
- **401 Unauthorized**: Invalid client ID or secret
- **403 Forbidden**: API client not authorized for the business
- **404 Not Found**: Wallet not found
