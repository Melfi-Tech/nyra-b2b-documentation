## Overview

Retrieves a specific customer sub-wallet for the business associated with the authenticated API client. This endpoint provides detailed information about a particular customer wallet including account details and balance information.

## Authentication

- **Type**: API Client
- **Headers**:
  - `x-client-id`: Client ID (required, string)
  - `Authorization`: `Bearer <client_secret>` (required, string)

## Prerequisites

- API client must be authenticated and authorized for the business
- Valid wallet ID must be provided
- Wallet must belong to the business

## Response

Returns detailed wallet information including account number, bank name, balance, and ownership details.

## Error Handling

- **401**: Invalid client ID or secret
- **403**: API client not authorized for the business
- **404**: Wallet not found
