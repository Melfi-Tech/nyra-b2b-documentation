## Overview

Retrieves all business fee configurations with cursor-based pagination. Supports filtering by business ID and date range for comprehensive fee management.

## Authentication

- **Type**: JWT Token
- **Headers**:
  - `Authorization`: `Bearer <jwt_token>` (required, string)
- **Required Role**: ADMIN or SUPER_ADMIN

## Query Parameters

All parameters are optional and support flexible filtering and pagination:

- **cursor**: Cursor for pagination (string)
- **from**: From date in ISO format (string, e.g., "2025-01-01")
- **to**: To date in ISO format (string, e.g., "2025-01-31")
- **page_size**: Number of results to return (number, default: 20)
- **lean**: Lean option for the query (boolean)

## Response

Returns an array of fee configurations with pagination metadata for efficient data retrieval.

## Pagination

This endpoint uses cursor-based pagination:
- Use the `cursor` parameter to get the next page of results
- The `meta.next_cursor` field contains the cursor for the next page
- When `next_cursor` is `undefined`, there are no more results

## Error Handling

- **401**: Invalid JWT token
- **403**: User doesn't have ADMIN or SUPER_ADMIN role
